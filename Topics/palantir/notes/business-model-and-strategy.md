# Palantirのビジネスモデルと戦略的示唆

## Summary
FDEモデル・Ontology・難市場選択の三位一体が、競合不可能な参入障壁を構築する設計になっている。投資家視点では「設計された赤字」として読む力が問われ、起業家視点では「スケールしないことの意図的選択」が長期独占の布石になるという示唆を持つ。

## Notes

### FDEモデルの本質

FDE（Forward Deployed Engineer）は単なる導入支援要員ではない。構造的には：

```
FDE派遣 → 現場課題の一次情報取得
       → プロダクトへのフィードバック（Ontology構築）
       → 顧客内でのユースケース拡張（Land）
       → 他部署・他機関への横展開（Expand）
```

**PLGの人間版**。プロダクトが自走できない複雑な領域で人間がその役割を担う。FDE期間（6〜18ヶ月）は「将来のスイッチングコストへの先行投資」として読むべきコストである。

### 難市場選択の論理

初期ターゲットは政府・諜報機関という最も参入障壁が高い市場：
- 調達プロセスが複雑・長期
- 守秘義務でケーススタディを語れない
- 規制・機密の壁が高い

これはThielの「競争を避けよ」思想の実践。誰も本気で取りに行かないから、突破後は独占できる。**最難市場を突破口にする逆張り設計**。

### 投資家視点からの示唆

**1. TAMは「今存在する市場」で測るな**
Palantirが2004年に狙った市場は当時ほぼ存在しなかった。「顧客がまだ自分の問題をソフトウェアで解けると思っていない市場」がブルーオーシャン。

**2. 赤字を「悪い赤字」と「良い赤字」で区別せよ**
- 悪い赤字：競合に負けている・オペレーションが非効率
- 良い赤字：顧客基盤深堀り・将来のスイッチングコスト構築中

Palantirの17年赤字は後者。政府契約のマルチイヤー・ノンキャンセラブルな性質が「将来収益の担保」として機能した。

**3. 創業者リスクは「思想家×実務家ペア」で見よ**
Thiel（何を作るか）とKarp（どう売り・守るか）の補完構造が機能の本質。一人の天才より思想家×実務家のペア構造を評価せよ。

### 起業家視点からの示唆

**パターン①：ビーチヘッド市場を極端に選ぶ**
難しい市場で勝てれば簡単な市場へ横展開できる。難しい市場の顧客は長期契約・解約しない。政府実績が民間展開の信頼性を担保した。

**パターン②：最初の10社を深くやり切る**
「最初の100社より最初の10社を深く」。FDEモデルは顧客との関係が深まるほど競合排除力が上がる指数的構造。

**パターン③：イデオロギーをプロダクトに組み込む**
- 採用：思想に共鳴する優秀人材を報酬以外の動機で集める
- 営業：「なぜあなたたちと仕事するか」の説明が不要になる
- 顧客選別：思想と合わない顧客を断ることがブランド価値を高める（中国政府案件拒否等）
